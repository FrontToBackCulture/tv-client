tv-client Architecture Reference

tv-client is the ThinkVAL desktop app — a Tauri 2 shell wrapping a React 18 single-page app, with a Rust backend for native capabilities and an embedded tv-mcp sidecar for MCP tools. It's the primary human UI for the ThinkVAL operating system.

This page is a visual map: what modules exist, how the shell is laid out, where data flows, and where to dig when you want to change something. For prescriptive guidance (patterns to follow, components to reuse), read CLAUDE.md at repo root, plus docs/dev/COMPONENTS.md, docs/dev/MECHANISMS.md, docs/dev/MODULES.md, and docs/dev/DESIGN_SYSTEM.md.

Stack

  • Shell: Tauri 2 (Rust + system webview), distributed as signed DMG (macOS) and MSI (Windows)
  • UI: React 18, TypeScript, Vite 6, Tailwind 3
  • State: Zustand 5 (client state), TanStack Query 5 (server state)
  • Data: Supabase JS (workspace DB), Tauri IPC (native commands), tv-mcp sidecar (MCP tools)
  • Editor: TipTap (rich text), Excalidraw (diagrams)
  • Auth: Supabase Auth PKCE → gateway JWT → workspace JWT (dual-project model)

Data Flow

Three data sources, three patterns. Don't mix them — each has its layer.

React UI
   │
   ├─► TanStack Query + Supabase JS  ───────►  Workspace Supabase
   │   (hooks in src/hooks/*)                   (business data: CRM, tasks, emails)
   │
   ├─► Tauri invoke(...)  ─────────────────►   Rust commands (src-tauri/src/commands/)
   │   (native: files, shell, OS, Outlook,      (heavy sync jobs, native APIs)
   │    Notion, S3, scheduler, etc.)
   │
   └─► MCP protocol  ───────────────────────►  tv-mcp sidecar binary
       (Claude Code actions inside the app)     (100+ tools — see tv-mcp-tools.html)

Auth & Workspaces

Two Supabase projects. tv-client talks to both:

  • Gateway Supabase — owns user identity, workspace registry, and JWT minting. Users log in once here via PKCE OAuth.
  • Workspace Supabase — one per workspace (e.g., ThinkVAL = cqwcaeffzanfqsxlspig). Holds all business data. tv-client gets a workspace JWT via the gateway-workspace-token edge function.

The Supabase client is a proxy — when the user switches workspace, the proxy hot-swaps its underlying client so existing React Query hooks keep working without re-mounting.

Hooks 54

Data-layer hooks in src/hooks/. Each wraps TanStack Query or a Tauri command and exposes a clean API to components. Hooks own queries, mutations, optimistic updates, and invalidation.

Organized into per-domain folders (apollo/, chat/, cockpit/, crm/, email/, feed/, gallery/, github-sync/, investment/, notion/, portal/, product/, prospecting/, public-data/, repos/, scheduler/, skills/, solutions/) plus ~25 top-level hooks (useCalendar, useDiscussions, useFavorites, useFiles, useNotifications, useOutlook, useLookupValues, etc.).

Rule of thumb: if a component needs data, it calls a hook. If the hook doesn't exist, create it — don't inline useQuery in components. See docs/dev/MECHANISMS.md for the full pattern.

Tauri Rust Backend Rust

Native code in src-tauri/src/. Each module under commands/ exposes Tauri commands that the React app calls with invoke(). Used for anything the webview can't do: filesystem, shell, OS APIs, long-running syncs, OAuth token exchange.

tv-mcp Sidecar

tv-mcp is not bundled in the app anymore (as of v0.10.30). Users install it separately to ~/.tv-mcp/bin/tv-mcp. tv-client doesn't spawn it — Claude Code does. tv-client only reads shared settings from ~/.tv-mcp/settings.json.

For the full list of 96 tools tv-mcp exposes, see tv-mcp-tools.html.
